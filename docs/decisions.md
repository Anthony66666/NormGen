# Decisions

- Prediction context membership, coordinate center, adaptive position scale, labels, and map ranking are functions of observed history only.
- Future availability is represented only by supervised loss masks; it is never used as conditioner context.
- Static length/width and scene normalization scale are conditioner metadata, not future channels generated by the flow; prediction defaults to five dynamic channels.
- The local epoch must use real INTERACTION records and the complete configured model architecture; scene limiting is permitted only to bound verification time.
- Prediction targets are per-step deltas of the five dynamic state channels, anchored at the last valid history state; saved samples and trajectory metrics use decoded absolute states.
- The prediction configuration uses one flow block over 30 future steps so the model does not require synthetic time padding.
- Coupling RoPE never interprets latent channels as coordinates. It uses last-valid history anchors, multiplied by each scene's position scale so relative distances are expressed in metres; the global scene centre is unnecessary because RoPE centres coordinates internally.
- FP32 remains the supported training path. AMP stays disabled until the RoPE SDPA attention-mask dtype is made FP16/BF16 compatible and separately validated.
- Training datasets, checkpoints, generated NPZ files, PNGs, TensorBoard events, and other `artifacts/` outputs are local validation evidence and are not committed to Git.
